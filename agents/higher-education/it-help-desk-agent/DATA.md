# Data Sources

Systems and platforms commonly accessed for campus IT support.

## IT Service Management (ITSM)

- **ServiceNow** -- incident management (incident number, short description, category, subcategory, priority: critical/high/medium/low, assignment group, assigned to, state: new/in progress/on hold/resolved/closed, opened date, resolved date, resolution notes, resolution code, customer satisfaction score), service requests (request number, catalog item, requested for, request details, approval status, fulfillment group, SLA target, SLA breach flag), change management (change request number, type: standard/normal/emergency, risk level, change window, CAB approval, implementation plan, backout plan, post-implementation review), knowledge base (article ID, title, category, content, view count, helpfulness rating, last updated, linked incidents, article owner), CMDB (configuration items: CI name, class, status, owner, location, serial number, asset tag, relationships, dependency maps), SLA tracking (SLA definition, target response time, target resolution time, actual response time, actual resolution time, breach status, percentage met)
- **Jira Service Management** -- ticket queues (issue key, summary, request type, priority, status, assignee, reporter, created date, updated date, resolution date, SLA time remaining), request types (request category, form fields, approval workflows, automation rules, linked knowledge articles), SLA management (SLA goals by priority/request type, time to first response, time to resolution, breach notifications, SLA reports), knowledge base (Confluence-linked articles, search analytics, article feedback, suggested articles for ticket types)
- **Freshservice** -- incident management (ticket ID, subject, type, status, priority, source: email/portal/phone/chat, agent, group, due date, resolution time, first response time, customer satisfaction), asset tracking (asset name, type, status, user, location, purchase date, warranty expiration, depreciation, vendor, serial number), change management (change type, risk, impact, approval board, scheduled dates, implementation status), service catalog (catalog items, categories, workflows, approval chains, fulfillment times, cost)
- **TeamDynamix (TDX)** -- IT service management (tickets: ID, title, type, status, priority, responsible group, responsible individual, created/modified dates, SLA status, time entries), project management (project name, status, timeline, tasks, milestones, resource allocation, budget, percent complete), asset management (asset ID, name, type, status, owner, location, purchase info, maintenance schedules, disposal date), client portal (self-service forms, knowledge articles, ticket submission, ticket status tracking, service catalog)
- **BMC Helix / Remedy** -- incident management (incident ID, summary, status, impact, urgency, priority matrix, assignment, SLA clock, work notes, resolution), problem management (problem ID, root cause, known error, related incidents, workaround, permanent fix, status), change management (change ID, class, type, risk level, schedule, approvals, implementation tasks, closure code)

## Identity & Access Management

- **Microsoft Entra ID (Azure AD)** -- user accounts (UPN, display name, email, department, job title, manager, account status: enabled/disabled, creation date, last sign-in date, license assignments), group memberships (security groups, Microsoft 365 groups, dynamic groups, group owners, member lists, nested groups), MFA status (MFA registration status, default MFA method: Authenticator app/SMS/phone call, registered methods, MFA enforcement policy, per-user MFA state), conditional access policies (policy name, state: enabled/disabled/report-only, conditions: users, apps, locations, device platforms, risk levels; controls: require MFA, compliant device, approved app), password resets (self-service password reset eligibility, reset method, reset date, administrator reset flag, password policy: expiration, complexity, lockout threshold)
- **Okta** -- SSO management (application catalog, SSO-enabled apps, SAML/OIDC configuration, sign-on policies, session settings), application access (app assignments: user/group, provisioning status, access request workflows, app usage analytics), MFA enrollment (enrolled factors: Okta Verify, SMS, email, FIDO2 WebAuthn, security questions; enrollment status, factor reset requests), lifecycle management (user provisioning, deprovisioning, suspension, reactivation, profile sync, joiner-mover-leaver workflows, HR-driven triggers)
- **CyberArk / BeyondTrust** -- privileged access management (privileged account inventory, session recording, access requests, approval workflows, check-in/check-out logs, credential rotation schedules), credential vaulting (stored credentials, access audit trail, password rotation policy, safe membership, authorized users)
- **Active Directory (on-prem)** -- user accounts (sAMAccountName, DN, OU placement, account status, last logon timestamp, password last set, account expiration, account lockout status), group policies (GPO name, linked OUs, settings: security, software deployment, scripts, folder redirection; enforcement, inheritance blocking), OU structure (organizational hierarchy, delegated administration, object counts by OU), password policies (minimum length, complexity requirements, history, maximum age, lockout threshold, lockout duration)

## Endpoint & Device Management

- **Microsoft Intune / SCCM** -- device enrollment (device name, OS, OS version, enrollment date, enrollment type: user/device, compliance status, last check-in, primary user), software deployment (application name, version, deployment type: required/available, installation status, deployment target, detection rules, supersedence), compliance policies (policy name, platform, settings: encryption, password, OS version, threat level; compliant/non-compliant/not-evaluated device counts, non-compliance reasons), remote wipe (wipe request date, wipe type: full/selective, status, initiated by, device affected)
- **JAMF Pro** -- macOS/iOS device management (device name, serial number, model, OS version, last check-in, enrollment type, managed status, FileVault status, Gatekeeper status), app deployment (app name, version, scope, installation method: self-service/automatic, VPP assignment, install status, update policies), security policies (configuration profiles: Wi-Fi, VPN, email, restrictions, passcode, disk encryption; profile scope, installation status), inventory (hardware details: model, processor, RAM, storage; installed software, certificates, local user accounts, extension attributes)
- **CrowdStrike / Carbon Black** -- endpoint detection and response (alert ID, severity: critical/high/medium/low/informational, detection name, tactic/technique MITRE ATT&CK mapping, host name, user, timestamp, process tree, network connections, file hash), threat alerts (alert type, threat score, IOC indicators, containment status, remediation actions), device health (sensor version, sensor status: online/offline/reduced functionality, prevention policy, last seen date, OS, device group)
- **Mosyle** -- Apple device management (enrollment status, device details, OS version, supervision status, DEP/ADE assignment), app management (managed apps, app versions, installation status, license usage, VPP token assignments), security compliance (passcode compliance, encryption status, software update compliance, security configuration profiles, lost mode status)

## Learning Management & Classroom Tech

- **Canvas / Blackboard / Brightspace / Moodle** -- LMS access issues (login failures, SSO errors, browser compatibility, mobile app issues, access permission errors, course enrollment discrepancies), integration troubleshooting (LTI tool connectivity, publisher platform integrations, SIS grade passback failures, API errors, plagiarism detection service status), user provisioning (SIS-driven account creation, role assignments, enrollment syncs, cross-listing, term setup, section creation errors)
- **Zoom** -- meeting setup (meeting ID, host, scheduled time, duration, settings: waiting room, password, recording, breakout rooms), recording access (cloud recording storage, download permissions, sharing settings, transcript availability, retention policy), SSO integration (SAML configuration, provisioning, deprovisioning, license assignment sync), license management (license type: Basic/Pro/Business, license assignment, usage statistics, concurrent meeting limits)
- **Panopto / Kaltura / YuJa** -- lecture capture (scheduled recordings, recording device, room assignment, processing status, duration, storage used), video hosting (video library, folder structure, permissions, sharing settings, embed codes, caption status), playback issues (buffering, codec compatibility, browser support, bandwidth requirements, DRM, device compatibility, closed caption sync)
- **Crestron / Extron** -- classroom AV systems (room name, system components: projector, display, camera, microphone, speakers; control processor status, firmware version), projector control (power state, input source, lamp hours, filter status, resolution, error codes), room scheduling panels (room availability display, booking integration, occupancy sensors, touch panel firmware, network connectivity)

## Network & Infrastructure

- **Cisco Meraki / Aruba** -- Wi-Fi access points (AP name, model, location, status: online/offline, client count, channel, power level, firmware version, uptime), network health (network utilization, throughput, latency, packet loss, error rates, top applications, top clients), client connectivity (client MAC, IP address, SSID, AP association, signal strength, data usage, connection duration, authentication method), RADIUS authentication (authentication requests, success/failure counts, failure reasons, NAS identifier, user identity, authentication latency)
- **Palo Alto / Fortinet** -- firewall rules (rule name, source zone, destination zone, source/destination addresses, application, service/port, action: allow/deny/drop, rule hit count, logging), VPN connectivity (VPN type: GlobalProtect/IPsec/SSL, user, connection status, IP assignment, connection duration, bandwidth usage, authentication method, connection failures), security policies (policy name, threat prevention profiles, URL filtering categories, file blocking, data loss prevention rules, security profile groups)
- **SolarWinds / Nagios / PRTG** -- network monitoring (node name, IP, status: up/down/warning, response time, packet loss, CPU utilization, memory utilization, interface statistics), uptime alerts (alert type, severity, trigger condition, node, interface, timestamp, acknowledgment status, escalation), bandwidth utilization (interface throughput: inbound/outbound, utilization percentage, top talkers, historical trends, capacity planning data)
- **Cloudflare / Akamai** -- DNS management (domain records: A, AAAA, CNAME, MX, TXT, SRV; TTL, proxy status, DNSSEC), CDN (cache hit ratio, bandwidth saved, origin requests, edge response time, purge requests, cache rules), DDoS protection (attack logs: type, duration, mitigated requests, origin, action taken; rate limiting rules, challenge pages), web application firewall (WAF rules, managed rulesets, custom rules, block/challenge/allow actions, event logs, false positive tuning)

## Email & Collaboration

- **Microsoft 365 / Exchange Online** -- email delivery (message trace: sender, recipient, subject, date, status: delivered/failed/deferred/quarantined, delivery time, message size), calendar issues (calendar sharing permissions, room mailbox bookings, scheduling conflicts, delegate access, free/busy publishing), Teams (team/channel creation, membership, guest access, meeting policies, app permissions, usage analytics), SharePoint (site collections, document libraries, permissions, storage usage, sharing settings, version history), OneDrive (storage quota, usage, sync status, sharing activity, external sharing policies, file recovery)
- **Google Workspace** -- Gmail (message delivery, spam filtering, quarantine, mail routing rules, send-as aliases, delegation, vacation responder, mail labels), Drive (storage usage by user, sharing settings, external sharing policies, file ownership, shared drives, DLP scan results), Calendar (resource calendars, booking rules, calendar sharing, event details, conference room equipment), Meet (meeting creation, recording, attendance reports, quality metrics, dial-in configuration), admin console (user management, organizational units, app access, security settings, audit logs, reports, alerts)
- **Slack** -- workspace administration (workspace settings, channel policies, message retention, file storage, app approval settings), channel management (channel creation, archival, membership, public/private, shared channels, channel activity metrics), app integrations (installed apps, bot tokens, webhook configurations, OAuth scopes, app usage statistics, security review status)

## Printing & Labs

- **PaperCut** -- print management (print queue name, printer location, printer status, job counts, page counts by user/department, color vs. B&W breakdown), quotas (user quota allocation, quota balance, quota period, top-up history, quota exceeded notifications), release stations (release station location, held jobs, release method: card swipe, PIN, mobile; release timeout, uncollected job counts), usage reporting (print volumes by user/department/printer, cost allocation, environmental impact: pages saved, trees saved, CO2 reduced)
- **Pharos** -- print queues (queue name, associated printers, queue status, pending jobs, default settings), mobile printing (print submission via email/web/app, job routing, release station assignment, held job count), cost recovery (cost per page: B&W/color/large format, user account balances, department charge-back, payment methods: student account, credit card, campus card)
- **LabStats** -- computer lab utilization (lab name, location, total stations, stations in use, stations available, utilization percentage, peak usage times, average session duration), software usage tracking (application name, launch count, total usage time, unique users, license compliance, underutilized software identification), availability dashboards (real-time lab availability by location, historical utilization patterns, heatmaps by time/day, capacity planning data)
