# Data Sources

Systems and platforms commonly accessed for enterprise IT support.

## IT Service Management (ITSM)

- **ServiceNow** -- incident management (incident ID, caller, category, subcategory, priority, impact, urgency, assigned group, assigned to, state, SLA, resolution notes, work notes, reopened count), service requests (request ID, catalog item, requested for, variables, approval chain, fulfillment group, SLA, status), change management (change ID, type, risk, impact, CAB approval, implementation plan, backout plan, schedule, status, post-implementation review), knowledge base (article ID, title, category, content, views, helpfulness rating, last updated, owner, review date), CMDB (CI name, CI class, serial number, IP address, relationships, discovery source, last discovered, operational status, supported by), SLA tracking (SLA definition, start time, breach time, elapsed time, pause conditions, met/breached), virtual agent (conversation ID, topic, intent, resolution, escalation, user satisfaction)
- **Jira Service Management** -- ticket queues (queue ID, issue type, priority, status, assignee, reporter, SLA, labels, components), request types (request type ID, name, fields, workflow, SLA, portal group, description), SLA management (SLA name, calendar, goal, time metric, conditions, breach notification), knowledge base (article ID, space, title, content, linked issues, views, helpful votes), automation rules (rule ID, trigger, conditions, actions, execution count, last triggered)
- **Freshservice** -- incident management (ticket ID, requester, type, status, priority, agent, group, SLA, satisfaction rating, tags), asset tracking (asset ID, asset type, serial number, assigned user, status, warranty, vendor, location), change management (change ID, type, risk, impact, planned dates, approval status, rollout plan), service catalog (catalog item, category, description, cost, fulfillment time, approval required, delivery group), AI-powered suggestions (ticket classification, agent recommendation, solution suggestion, similar tickets, knowledge article match)
- **BMC Helix / Remedy** -- incident management (incident ID, classification, urgency, impact, priority, assigned group, owner, status, resolution, root cause), problem management (problem ID, related incidents, root cause analysis, known error, workaround, permanent fix, status), change management (change ID, class, risk level, impact, schedule, approvals, implementation status), self-service portal (request catalog, knowledge search, chat, status tracking, announcements)
- **Zendesk** -- ticket management (ticket ID, requester, subject, description, status, priority, assignee, group, tags, custom fields, satisfaction rating), macros (macro ID, title, actions, usage count, availability scope), SLA policies (SLA name, target, metric, conditions, business hours, breach notification), help center (article ID, section, title, body, labels, views, votes, comments, translation), AI assist (suggested responses, ticket classification, sentiment analysis, intent detection)

- **Okta** -- SSO management (app ID, app name, sign-on mode, authentication policy, assigned users/groups, provisioning status), application access (app assignment, profile mapping, attribute mapping, provisioning events, deprovisioning triggers), MFA enrollment (factor type, enrollment status, last verified, provider, phone number), lifecycle management (lifecycle state, status, created date, activated date, last login, password changed, suspended), universal directory (profile attributes, custom attributes, attribute mappings, import sources)
- **Ping Identity** -- SSO (application federation, SAML/OIDC configuration, session settings, authentication policy, sign-on URL), adaptive MFA (risk score, authentication step-up, device trust, geolocation, behavior analytics), directory integration (directory type, attribute sync, provisioning rules, group sync, schema mapping), API security (API gateway, token management, rate limiting, OAuth policies, access tokens)
- **CyberArk** -- privileged access management (safe, account, platform, target system, last password change, compliance status), credential vaulting (credential ID, account name, platform, safe, password age, rotation policy, last reconciliation), session recording (session ID, user, target, duration, keystrokes, commands, risk events, recording URL)
- **SailPoint** -- identity governance (identity ID, identity attributes, entitlements, roles, access reviews, certification campaigns, risk score), access certifications (campaign ID, certifier, items to review, decisions, completion percentage, due date), role management (role ID, name, entitlements, membership rules, owner, risk level, segregation-of-duty conflicts), lifecycle automation (joiner/mover/leaver events, triggered workflows, provisioning actions, deprovisioning actions, audit trail)

- **JAMF Pro** -- macOS/iOS management (device ID, serial number, model, OS version, last check-in, managed status, enrolled via), app catalog (app name, version, scope, distribution method, install status, VPP license), security policies (policy name, payload, scope, frequency, self-service, exclusions), inventory (hardware specs, storage, installed apps, certificates, local accounts, FileVault status, Gatekeeper status), self-service (app catalog, policies, bookmarks, categories, user eligibility, install history)
- **VMware Workspace ONE** -- unified endpoint management (device ID, platform, enrollment type, compliance state, last seen, user), app distribution (app name, version, assignment, install method, status, store), compliance (compliance policy, rules, device state, remediation, escalation, exemptions)
- **CrowdStrike Falcon** -- endpoint detection and response (detection ID, severity, technique, tactic, hostname, user, timestamp, disposition), threat intelligence (IOC, threat actor, campaign, severity, affected hosts, recommended action), device health scoring (device ID, score, contributing factors, last assessed, sensor version, policy group)
- **Tanium** -- endpoint visibility (endpoint ID, hostname, IP, OS, patch level, installed software, running processes, logged-in users), patch management (patch ID, KB article, severity, applicable devices, installed devices, missing devices, compliance percentage), vulnerability assessment (CVE ID, severity, CVSS score, affected endpoints, remediation status, exploitability)
