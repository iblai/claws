# Data Sources

Systems and platforms commonly accessed for enterprise knowledge retrieval.

## Enterprise Search

- **Glean** -- unified search across SaaS applications (query, results, source application, relevance score, permissions-aware filtering, personalization, result type), AI-powered answers (query, generated answer, source documents, citations, confidence score, feedback rating), people search (name, title, department, expertise areas, contact info, org chart position, recent documents, collaboration patterns), content recommendations (recommended content, source, recency, relevance, user activity context, trending topics)
- **Coveo** -- relevance platform (query pipeline, ranking expressions, featured results, query suggestions, machine learning models, A/B test results), AI search (query, results, ML-ranked relevance, automatic relevance tuning, query understanding, intent detection), content recommendations (recommendation model, strategy, items, click-through rate, personalization context), usage analytics (query volume, top queries, zero-result queries, click-through rate, search success rate, content gaps, time-to-result)
- **Elastic Enterprise Search** -- full-text search (index, query, results, relevance score, highlights, facets, filters, aggregations), relevance tuning (boost rules, synonym sets, curations, precision metrics, recall metrics, click analytics), analytics (search volume, popular queries, failed searches, click-through rate, average click position, query latency), content connectors (source type, connector ID, sync schedule, indexed document count, last sync, field mappings, permissions sync)
- **Microsoft Search** -- cross-Microsoft 365 search (query, result type, source service, rank, file/page/message/person, permissions, bookmarks, Q&A, floor plans), SharePoint results (site, library, document, metadata, modified date, author, content type), OneDrive results (file, folder, owner, shared with, modified date, file type), Teams results (message, channel, team, sender, date, attachments), Outlook results (email, sender, recipients, date, attachments, folder)
- **Google Cloud Search** -- search across Google Workspace (query, results, source, rank, document type, owner, shared with, last modified), third-party connectors (connector ID, data source, indexed items, schema, access control, sync status, query federation)

## Knowledge Bases & Wikis

- **Confluence (Atlassian)** -- team spaces (space key, name, type, homepage, description, permissions, categories), knowledge articles (page ID, title, body, labels, version, author, created date, last modified, parent page, child pages, comments, likes), templates (template ID, name, description, body, labels, space scope, usage count), page trees (hierarchy, depth, sibling pages, navigation path), comments (comment ID, author, body, date, parent comment, inline/footer, resolved status)
- **Notion** -- team wikis (workspace, top-level pages, hierarchy, permissions, sharing, guest access), databases (database ID, title, properties, property types, views, filters, sorts, relations, rollups, formulas, automations), project documentation (page ID, content blocks, sub-pages, synced blocks, embeds, mentions, comments, last edited by, created time), integrations (connected tools, sync status, linked databases, API connections)
- **SharePoint** -- document libraries (library ID, columns, content types, views, versioning, co-authoring, check-in/out, metadata navigation), site pages (page ID, title, layout, web parts, audience targeting, publishing status, analytics), managed metadata (term store, term groups, term sets, terms, labels, synonyms, deprecation, translations), hub sites (hub ID, associated sites, navigation, search scope, theme, shared permissions)
- **Guru** -- verified knowledge cards (card ID, title, content, board, collection, verification status, verifier, last verified, next verification, trust score, tags), expert-owned content (expert ID, owned cards, domain, verification cadence, overdue items, quality score), browser extension (triggered cards, auto-suggestions, dismissed, clicked, search queries, context source), Slack/Teams integration (trigger phrases, suggested cards, click-through rate, helpful/not helpful, unanswered questions)
- **Tettra** -- internal knowledge base (page ID, title, category, content, author, created date, last updated, views, stale flag), Q&A (question ID, asker, question text, answer, answerer, date, upvotes, linked page, status), stale content detection (page ID, last updated, last viewed, staleness score, assigned reviewer, review deadline), Slack-first design (Slack triggers, auto-responses, unanswered queue, escalation, knowledge gap identification)
- **Slab** -- knowledge base (post ID, title, content, topic, author, created date, last updated, views, reactions), topic organization (topic ID, name, posts, sub-topics, owner, description, pinned posts), integrations (connected tools, synced content, linked references, search scope), unified search (query, results across integrations, source, relevance, recency, access level)

## Document Repositories

- **Google Drive** -- documents (doc ID, title, type, owner, editors, viewers, last modified, starred, folder path, sharing link), spreadsheets (sheet ID, title, sheets, named ranges, data validation, charts, pivot tables, connected sources), presentations (presentation ID, title, slides, speaker notes, layout, theme, shared with), shared drives (drive ID, name, members, roles, storage used, file count, organizational unit, activity log)
- **OneDrive / SharePoint Document Libraries** -- file storage (file ID, name, path, size, created date, modified date, author, version, shared with, sensitivity label), co-authoring (active editors, last edit time, edit session, comments, suggestions, version history), version history (version number, modified by, date, size, comments, restore option), metadata (columns, content type, managed metadata, custom properties, classification, retention label)
- **Box** -- cloud content management (file ID, name, path, size, version, owner, shared link, watermark, classification, description), workflows (workflow ID, trigger, steps, approvals, tasks, deadlines, status, completion), metadata (template, attributes, cascade policy, classification, custom metadata, AI-extracted metadata), governance (retention policy, legal hold, disposition, access stats, security classification, external collaboration restrictions), external collaboration (collaboration ID, collaborator, role, status, accepted date, company, access level)
- **Dropbox Business** -- file sync (file ID, name, path, size, hash, modified date, sync status, local/cloud, shared folder), Paper documents (doc ID, title, content, sections, tasks, mentions, comments, sharing, folder), team spaces (team folder, members, policies, storage, activity, admin controls), audit logs (event type, user, timestamp, file/folder, action, IP address, device, location)

## Communication Archives (as knowledge sources)

- **Slack** -- channel history (channel ID, name, type, messages, message count, members, topic, purpose, created date, archived status), threads (parent message, replies, reply count, participants, last reply date, subscribed users), saved items (item type, message/file reference, channel, date saved, user), canvas (canvas ID, title, content blocks, sharing, channel attachment, last edited), search across workspaces (query, results, source workspace, channel, sender, date range, file type, relevance)
- **Microsoft Teams** -- chat history (chat ID, type, participants, messages, last activity, read receipts, pinned messages), channel messages (team ID, channel ID, message ID, sender, content, reactions, replies, mentions, attachments, importance), meeting transcripts (meeting ID, transcript text, speakers, timestamps, action items, AI-generated notes), wiki tabs (wiki ID, team, channel, sections, pages, content, last modified, contributors)
- **Loom** -- video library (video ID, title, creator, duration, created date, views, folder, workspace, privacy, password), transcripts (video ID, transcript text, speakers, timestamps, searchable segments, auto-chapters), search across recordings (query, matched videos, matched segments, timestamp, speaker, relevance score)
- **Yammer / Viva Engage** -- community discussions (community ID, name, type, members, posts, replies, reactions, trending topics, pinned posts), Q&A (question ID, author, question text, answers, best answer, upvotes, views, resolved status), knowledge sharing (post type, attachments, links, hashtags, mentions, reactions, shares, reach, impressions)

## Developer Knowledge

- **GitHub / GitLab** -- code repositories (repo name, description, language, stars, forks, license, README, last updated, default branch, topics), READMEs (content, structure, badges, installation instructions, usage examples, contributing guidelines), wikis (wiki pages, titles, content, last updated, contributors, sidebar navigation), issues (issue ID, title, body, labels, assignee, milestone, state, comments, reactions), pull request discussions (PR ID, title, description, reviewers, comments, review decisions, linked issues, diff summary)
- **Stack Overflow for Teams** -- internal Q&A (question ID, title, body, tags, author, date, score, views, answers, accepted answer, comments), articles (article ID, title, content, tags, author, date, views, bookmarks), collections (collection ID, name, questions, articles, followers, description), tags (tag name, count, description, related tags, top contributors, wiki excerpt)
- **Backstage (Spotify)** -- developer portal (catalog entities, components, APIs, systems, domains, resources, groups, users), service catalog (component name, type, owner, lifecycle, system, description, tags, dependencies, API specs), tech docs (doc site, content, navigation, search, last updated, owner team, related component), API references (API name, type, definition URL, owner, system, lifecycle, consumers, documentation)

## CRM & Business Data (read-only context)

- **Salesforce** -- account records (account ID, name, industry, type, owner, annual revenue, employees, address, tier, last activity), case history (case ID, account, contact, subject, description, status, priority, resolution, created date, closed date), knowledge articles (article ID, title, body, category, version, last published, views, vote score, URL name), reports (report ID, name, type, folder, columns, filters, groupings, last run, scheduled)
- **HubSpot** -- contact records (contact ID, name, email, company, lifecycle stage, lead status, owner, last activity, properties), deal history (deal ID, name, stage, amount, close date, owner, pipeline, associated contacts, deal type, activity timeline), knowledge base articles (article ID, title, body, category, subcategory, language, views, helpful votes, related articles)
- **ServiceNow** -- knowledge articles (article ID, title, body, knowledge base, category, workflow state, author, published date, views, helpful percentage, attachments), incident patterns (category, subcategory, frequency, common resolution, average resolution time, assignment group, related knowledge), CMDB data (CI class, CI name, attributes, relationships, operational status, support group, discovery source, environment)

## Analytics

- **Tableau / Power BI / Looker** -- published dashboards (dashboard ID, title, views, description, owner, workbook/report, data sources, last refreshed, scheduled refresh), report catalogs (report name, type, folder, description, fields, filters, owner, usage count, last accessed), data dictionaries (field name, description, data type, source table, calculation, business definition, steward)
- **Amplitude / Mixpanel** -- product analytics (event name, event properties, user properties, count, unique users, trends, segmentation), user behavior (user flows, session data, feature usage, retention, engagement, power users, drop-off), funnel data (funnel name, steps, conversion rate, drop-off rate, time to convert, segmentation, statistical significance)
